4.2 Interactive Voice Response.

 

An Interactive Voice Response (IVR) endpoint provides access to an IVR service. Under requests from the Call Agent, the IVR server will "play"announcements and tones, and will "listen" voice messages, from the user. A given IVR endpoint is not expected to support more than one connection at a time. If several connections were established to the same endpoint, then the same tones and announcements would be played simultaneously over all the connections.

 

4.2.1 Configuration.

 

The configurable attributes of the announcment player are as follows: 

  • JndiName - The JNDI name under which endpoint is to be bound.
  • BindAddress - The IP address to which this endpoint will be bound.
  • Jitter - The size of jitter buffer in milliseconds.
  • PacketizationPeriod - the period of media stream packetization in milliseconds.
  • PCMA - enable or disable G711 (A-law) codec support.
  • PCMU - enable or disable G711 (U-law) codec support.

 

4.2.2 Supported media types and formats 

 

Supported the media types and formats are listed below:

 

Media type: WAVE(.wav)  

  • 16-bit mono/stereo linear

4.2.4 DTMF Configuration.

The DTMF configuration is determinted by DTMF attribute. The properties are as follows:

  • detector.mode. Configures DTMF detector mode. Possible values are AUTO, INBAND and RFC2833.
  • dtmf.payload. Configures the payload number if RFC2833 mode is specified
